### 🎯 Key Insights About WordPress Menu Management via API:
- Menu items must be explicitly created as separate entities from posts
- The relationship is: Post → Menu Item (post_type) → Menu Position
- Hierarchical menus use parent-child relationships between menu items
- menu_order controls sequential positioning within the same parent level
- Changes may require cache clearing to appear immediately in UI
- Direct ID access is reliable when collection queries might be affected by filtering/caching

### ⚠️ Important Limitations Discovered:
- Standard WordPress REST API does NOT expose menu management endpoints (/wp-json/wp/v2/menus/) without proper authentication
- Menu management requires authenticated requests with sufficient permissions
- Even with super_admin privileges, some endpoints may behave unexpectedly (e.g., menu_order updates not reflecting immediately)
- Verification through multiple methods (direct fetch + collection query) is recommended for critical changes
